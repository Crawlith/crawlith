# Gemini CLI Integration for Crawlith Project

This document outlines the integration and usage of the Gemini CLI within the Crawlith project. The Gemini CLI is an interactive agent designed to assist with various software engineering tasks, adhering to project conventions and best practices.

## Agent Role and Capabilities

The Gemini CLI acts as a development assistant capable of:
- **Bug Fixing:** Identifying and resolving software defects.
- **Feature Development:** Assisting in the implementation of new features.
- **Code Refactoring:** Improving code structure without changing external behavior.
- **Code Explanation:** Providing insights into existing code.
- **Testing:** Writing and verifying unit and integration tests.
- **Codebase Investigation:** Analyzing project architecture and dependencies.
- **Responding to queries:** Answering questions related to the project or general software engineering.

## Invoking the Agent

The Gemini CLI is typically invoked from the command line in the project's root directory. Specific invocation commands may vary based on the environment and setup.

## Project Conventions and Adherence

The agent is configured to rigorously adhere to the following project conventions:
- **Code Style:** Mimicking existing formatting, naming, and structural patterns.
- **Frameworks and Libraries:** Verifying established usage before employing new ones.
- **Idiomatic Changes:** Ensuring changes integrate naturally with the existing codebase.
- **Testing:** Prioritizing the creation of unit tests for new features and bug fixes.
- **Git Workflow:** Avoiding staging or committing changes unless explicitly instructed.
