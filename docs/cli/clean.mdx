# clean

The `clean` command is a maintenance utility that helps you manage disk space and database size by removing old crawl data, exported reports, and snapshots. It offers granular control, allowing you to delete specific snapshots, clear all exports, or remove an entire site's history.

---

## Usage

```bash
crawlith clean <url> [options]
```

---

## Options

### `--exports`

**Type:** boolean
**Default:** `false`
**Required:** No

Deletes all exported reports and files (HTML, JSON, CSV) for the specified site from the output directory.

**Why It Matters for SEO**
Frees up valuable disk space by removing outdated HTML reports or large data exports that are no longer relevant to your current audit.

### `--db`

**Type:** boolean
**Default:** `false`
**Required:** No

Deletes all database records (snapshots, pages, links, metrics) for the specified site.

**Why It Matters for SEO**
Completely removes all historical data for a site from the internal database. This is useful when retiring a project, fixing a corrupted state, or resetting for a fresh, clean start.

### `--snapshot`

**Type:** number (id)
**Default:** `null`
**Required:** No

Deletes a single, specific snapshot by its ID.

**Why It Matters for SEO**
Allows you to surgically remove a failed, interrupted, or anomalous crawl without deleting the entire history of the site.

### `--output`

**Type:** string
**Default:** `./crawlith-reports`
**Required:** No

Specifies the output directory where exports are stored.

**Why It Matters for SEO**
Ensures the command targets the correct folder when deleting exported files, especially if you use a custom output path for your reports.

### `--yes`

**Type:** boolean
**Default:** `false`
**Required:** No

Skips the interactive confirmation prompt.

**Why It Matters for SEO**
Enables the use of this command in automated scripts, cron jobs, or CI/CD cleanup tasks where manual interaction is not possible.

---

## Example

```bash
crawlith clean https://example.com --exports --yes
```

This command deletes all exported files for `https://example.com` immediately, without asking for user confirmation.

---

## When To Adjust This

- **Regular Maintenance:** Use `--exports` periodically to prune old reports and keep your filesystem clean.
- **Database Hygiene:** Use `--snapshot` to remove incomplete or test crawls that skew your historical data.
- **Full Reset:** Use `--db` and `--exports` together to completely wipe a site's presence from Crawlith.
