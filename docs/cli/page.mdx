# page

The `page` command performs a deep, single-page analysis of a specific URL. It evaluates key on-page SEO factors, content quality metrics, and accessibility compliance without crawling the entire site. It is designed for rapid iteration during content development, specific page troubleshooting, or verification of fixes.

---

## Usage

```bash
crawlith page <url> [options]
```

---

## Options

### `--live`

**Type:** boolean
**Default:** `false`
**Required:** No

Forces a live HTTP request to fetch the page content instead of using cached data from a previous crawl.

**Why It Matters for SEO**
Essential for verifying immediate changes after deployment or content updates. It ensures you are analyzing the current state of the page as a user or search engine bot would see it, rather than stale data.

### `--export`

**Type:** string
**Default:** `false`
**Required:** No

Automatically exports the analysis results in the specified formats (comma-separated: `json`, `markdown`, `csv`, `html`).

**Why It Matters for SEO**
Generate instant reports for stakeholders. Share an HTML summary of a landing page's health or a JSON data object for integration with other monitoring tools.

### `--format`

**Type:** string (`pretty` | `json`)
**Default:** `pretty`
**Required:** No

Controls the output format of the CLI.

**Why It Matters for SEO**
`json` output provides raw data for automated validation scripts, while `pretty` offers a structured, readable summary for manual review in the terminal.

### `--seo`

**Type:** boolean
**Default:** `false`
**Required:** No

Filters the output to show only the SEO module results (meta tags, title, headers, canonicals).

**Why It Matters for SEO**
Focus your attention purely on technical metadata without distraction from content or accessibility metrics.

### `--content`

**Type:** boolean
**Default:** `false`
**Required:** No

Filters the output to show only the content module results (word count, readability, text-to-HTML ratio).

**Why It Matters for SEO**
Isolate content quality metrics to ensure the page meets editorial standards and provides sufficient value to search engines.

### `--accessibility`

**Type:** boolean
**Default:** `false`
**Required:** No

Filters the output to show only the accessibility module results (alt text, ARIA attributes).

**Why It Matters for SEO**
Accessibility is a ranking factor and crucial for user experience. This flag helps you specifically audit compliance with WCAG standards.

### `--proxy`

**Type:** string (url)
**Default:** `undefined`
**Required:** No

Sets a proxy server for the live request.

**Why It Matters for SEO**
Test how the page appears to users in different geographic locations or bypass IP-based restrictions.

### `--ua`

**Type:** string
**Default:** `undefined`
**Required:** No

Sets a custom User-Agent string for the request.

**Why It Matters for SEO**
Simulate Googlebot (smartphone or desktop) to verify that the server serves the correct content to crawlers and check for cloaking or rendering issues.

### `--max-redirects`

**Type:** number
**Default:** `2`
**Required:** No

Sets the maximum number of redirect hops to follow.

**Why It Matters for SEO**
Detect redirect chains that dilute link equity or cause latency. A low limit ensures you catch excessive hops early.

### `--cluster-threshold`

**Type:** number
**Default:** `10`
**Required:** No

Sets the similarity sensitivity for duplicate content detection against the internal database.

**How It Works**
Defines how strictly the page is compared to other known pages.
- **Lower value:** Requires near-identical content to be flagged as a duplicate.
- **Higher value:** Allows broader matching (e.g., similar templates).

**Why It Matters for SEO**
Useful when checking if a new landing page is sufficiently unique compared to existing pages on the site to avoid cannibalization.

---

## Example

```bash
crawlith page https://example.com/about --live --seo --format json
```

This command fetches the live version of `https://example.com/about`, analyzes only its SEO metadata, and outputs the result as a JSON object.

---

## When To Adjust This

- **Content Creation:** Use `--content` and `--live` while writing to check readability and keyword usage in real-time.
- **Pre-Commit Hook:** Use `--format json` to validate critical pages (like the homepage) before code commits.
- **Geo-Targeting:** Use `--proxy` to verify localized content delivery.
- **Mobile Audit:** Use `--ua` with a mobile user agent string to check mobile-first indexing compliance.
