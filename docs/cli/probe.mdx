# probe

The `probe` command performs a deep diagnostic analysis of the transport and network layer for a specific URL. It inspects SSL/TLS certificates, HTTP headers, negotiation protocols, and security configurations to ensure the site is served securely and efficiently.

---

## Usage

```bash
crawlith probe <url> [options]
```

---

## Options

### `--verbose`

**Type:** boolean
**Default:** `false`
**Required:** No

Prints a structured, human-readable report with expanded details about the connection.

**Why It Matters for SEO**
Allows you to verify the full certificate chain, expiration dates, and detailed handshake information, ensuring trust signals are valid for search engines.

### `--debug`

**Type:** boolean
**Default:** `false`
**Required:** No

Prints low-level timing, negotiation, and raw protocol details.

**Why It Matters for SEO**
Crucial for diagnosing connection latency, handshake failures, or protocol downgrades (e.g., falling back to HTTP/1.1 instead of HTTP/2), which can impact Core Web Vitals.

### `--json`

**Type:** boolean
**Default:** `false`
**Required:** No

Outputs the results as a structured JSON object.

**Why It Matters for SEO**
Enables the integration of transport security and performance checks into automated monitoring systems and CI/CD pipelines.

### `--timeout`

**Type:** number (ms)
**Default:** `10000`
**Required:** No

Sets the request timeout in milliseconds.

**Why It Matters for SEO**
Helps test how the server handles slow connections or heavy load, simulating poor network conditions often encountered by mobile users.

---

## Example

```bash
crawlith probe https://example.com --verbose
```

This command inspects `https://example.com` and prints a detailed report of its SSL/TLS configuration, headers, and connection properties.

---

## When To Adjust This

- **Security Audit:** Use `--verbose` to check for weak ciphers, expired certificates, or missing security headers.
- **Latency Debugging:** Use `--debug` to pinpoint exactly which part of the connection (DNS, TCP, TLS) is causing delays.
- **Automation:** Use `--json` to pipe results into a logging system or dashboard.
